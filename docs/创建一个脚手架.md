# 一、脚手架是什么？

在编程里，**脚手架 = 项目快速生成工具**

我用最直白的话给你讲清楚：就是一个**自动帮你搭好项目架子的工具**。

你不用从零写配置、建文件夹、装依赖，一行命令就能生成一个**能直接跑的完整项目结构**。



## 举个最常见的例子

- **Vue 脚手架：** `vue-cli` / `create-vue`
- **React 脚手架：** `create-react-app`
- **NestJS 脚手架：** `nest-cli`
- **Java SpringBoot 脚手架：** `Spring Initializr`

你敲一行命令：

```bash
vue create my-project
```

它就自动帮你建好：

- 文件夹结构
- 配置文件（babel、eslint、vite/webpack）
- 基础依赖
- 入口文件、示例页面
- 运行脚本（dev /build）

你**直接写业务代码就行**，不用管环境怎么搭。

## 生活中的比喻

- **盖房子**：脚手架 = 搭好的钢架结构
- **写项目**：脚手架 = 帮你搭好项目结构的工具

你只需要往里面 “砌墙、装修”，不用从零打地基。

## 一句话总结

脚手架 = 快速创建项目模板的工具

帮你省去环境配置，直接开始写业务。

你现在看到的 `vite`、`create-vue`、`nest new` 全都是脚手架。



# 二、为什么要定制脚手架

为什么公司 / 开源项目要自己定制脚手架？

而不是直接用官方的 vue-cli /create-vue？

定制脚手架 = 把公司所有标准、规范、模板、权限、布局、组件，全部打包成一个 “一键生成工具”。

## 1. 统一团队规范

公司里每个人写代码风格不一样。

定制脚手架可以强制：

- 统一目录结构
- 统一 ESLint 规范
- 统一 Git 提交规范
- 统一权限、路由、请求封装
- 统一主题、布局、组件库

**所有人新项目长得一模一样，不会乱。**

## 2. 省去重复搭建

每次新项目都要：

- 装 Vue
- 装 Axios
- 装 Pinia
- 配路由
- 配权限
- 配请求拦截
- 配 ESLint
- 配环境变量
- 配打包配置
- 配布局框架
- 配登录页面

**定制脚手架 = 一行命令全部搞定。**

## 3. 内置企业通用功能

像 **@pureadmin/cli** 这种脚手架，

**内置了企业后台 80% 的通用功能：**

- 登录
- RBAC 权限
- 动态路由
- 按钮权限
- 字典管理
- 布局系统
- 请求封装
- 多主题
- 多语言

**新项目不用从零写，直接二次开发。**

## 4. 降低新人上手成本

新人来了，不用教架构，

**直接运行脚手架，生成标准项目，马上能写业务。**



# 三、开发一个基础脚手架

脚手架 = **命令行工具 + 项目模板仓库**

- 命令行工具：负责交互、拉取模板、生成项目、安装依赖
- 模板仓库：存你标准化的项目架子（Vue3+TS+Element Plus 后台模板等）

## 技术选型

- **Node.js + Commander.js**：处理命令行参数、交互
- **Inquirer.js**：做交互式选择（选模板、选功能）
- **chalk + ora**：美化终端输出（彩色日志、加载动画）
- **download-git-repo**：从 GitHub/Gitee 拉取模板
- **fs-extra**：处理文件写入、重命名
- **cross-spawn**：跨平台执行 `npm install` 等命令

## 完整实现步骤

### 1. 初始化脚手架项目

```bash
# 1. 创建脚手架项目目录
mkdir my-cli && cd my-cli

# 2. 初始化 package.json（一路回车）
npm init -y

# 3. 安装核心依赖
npm install commander inquirer chalk ora download-git-repo fs-extra cross-spawn --save
```

### 2. 配入口文件与命令

在根目录创建 `bin/index.js`（脚手架入口）：

```js
#!/usr/bin/env node

const { Command } = require('commander');
const inquirer = require('inquirer');
const chalk = require('chalk');
const ora = require('ora');
const download = require('download-git-repo');
const fs = require('fs-extra');
const path = require('path');
const spawn = require('cross-spawn');

const program = new Command();

// 定义脚手架版本、命令
program
  .name('my-cli')
  .version('1.0.0')
  .description('我的自定义前端脚手架');

// 核心命令：my-cli create <项目名>
program
  .command('create <projectName>')
  .description('创建一个新的前端项目')
  .action(async (projectName) => {
    // 1. 交互式选择模板
    const { template } = await inquirer.prompt([
      {
        type: 'list',
        name: 'template',
        message: '请选择项目模板：',
        choices: [
          { name: 'Vue3+TS+Element Plus 后台管理模板', value: 'vue-admin' },
          { name: 'Vue3+TS+H5 移动端模板', value: 'vue-h5' },
          { name: 'React+TS+Ant Design 后台模板', value: 'react-admin' }
        ]
      }
    ]);

    // 2. 定义模板对应的 Git 仓库地址（换成你自己的模板仓库）
    const templateMap = {
      'vue-admin': 'github.com:your-username/vue3-admin-template#main',
      'vue-h5': 'github.com:your-username/vue3-h5-template#main',
      'react-admin': 'github.com:your-username/react-admin-template#main'
    };

    const repoUrl = templateMap[template];
    const projectPath = path.resolve(process.cwd(), projectName);

    // 3. 检查项目目录是否已存在
    if (fs.existsSync(projectPath)) {
      console.log(chalk.red(`❌ 目录 ${projectName} 已存在，请更换项目名！`));
      return;
    }

    // 4. 拉取模板（加载动画）
    const spinner = ora('🚀 正在拉取项目模板...').start();
    download(repoUrl, projectPath, { clone: false }, (err) => {
      spinner.stop();
      if (err) {
        console.log(chalk.red(`❌ 模板拉取失败：${err.message}`));
        return;
      }

      // 5. 替换项目名（修改 package.json 等文件）
      const pkgPath = path.join(projectPath, 'package.json');
      const pkg = fs.readJsonSync(pkgPath);
      pkg.name = projectName;
      fs.writeJsonSync(pkgPath, pkg, { spaces: 2 });

      console.log(chalk.green(`✅ 项目 ${projectName} 创建成功！`));

      // 6. 自动安装依赖
      const { install } = await inquirer.prompt([
        {
          type: 'confirm',
          name: 'install',
          message: '是否自动安装依赖？',
          default: true
        }
      ]);

      if (install) {
        const installSpinner = ora('📦 正在安装依赖...').start();
        const child = spawn('npm', ['install'], { cwd: projectPath, stdio: 'inherit' });
        child.on('close', (code) => {
          installSpinner.stop();
          if (code === 0) {
            console.log(chalk.green(`✅ 依赖安装完成！`));
            console.log(chalk.blue(`👉 启动项目：cd ${projectName} && npm run dev`));
          } else {
            console.log(chalk.red(`❌ 依赖安装失败，请手动执行 npm install`));
          }
        });
      } else {
        console.log(chalk.blue(`👉 请手动安装依赖：cd ${projectName} && npm install`));
      }
    });
  });

// 解析命令行参数
program.parse(process.argv);
```

### 3. 配置 `package.json`，绑定命令

```json
{
  "name": "my-cli",
  "version": "1.0.0",
  "description": "我的自定义前端脚手架",
  "main": "index.js",
  "bin": {
    "my-cli": "./bin/index.js"
  },
  "scripts": {},
  "dependencies": {
    "chalk": "^4.1.2",
    "commander": "^11.1.0",
    "cross-spawn": "^7.0.3",
    "download-git-repo": "^3.0.2",
    "fs-extra": "^11.2.0",
    "inquirer": "^8.2.6",
    "ora": "^5.4.1"
  }
}
```

### 4. 本地测试脚手架

```fs
# 1. 全局安装脚手架（把 my-cli 命令挂载到系统）
npm link

# 2. 测试命令
my-cli create my-test-project
```

如果能正常弹出选择、拉取模板、安装依赖，就说明脚手架跑通了！

## 核心功能拓展

### 1. 模板仓库怎么准备？

- 把你标准化的项目（比如 Vue3+TS+Element Plus 后台）推到 GitHub/Gitee
- 模板里要包含：统一的目录结构、ESLint 规范、请求封装、权限系统、布局等
- 脚手架拉取后，直接就是你公司的标准项目

### 2. 进阶功能

- **自定义配置项**：比如选择是否开启国际化、Tailwind CSS、Mock 数据
- **模板版本管理**：支持选择模板版本（稳定版 / 开发版）
- **Git 初始化**：自动执行 `git init`，关联远程仓库
- **环境变量配置**：自动生成 `.env.development`、`.env.production`
- **代码格式化**：安装依赖后自动执行 `npm run lint` 格式化代码

### 3. 发布到 npm

```shell
# 1. 登录 npm（如果没账号先注册）
npm login

# 2. 发布包
npm publish
```
