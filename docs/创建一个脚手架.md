# 一、脚手架是什么？

在编程里，**脚手架 = 项目快速生成工具**

我用最直白的话给你讲清楚：就是一个**自动帮你搭好项目架子的工具**。

你不用从零写配置、建文件夹、装依赖，一行命令就能生成一个**能直接跑的完整项目结构**。

## 举个最常见的例子

- **Vue 脚手架：** `vue-cli` / `create-vue`
- **React 脚手架：** `create-react-app`
- **NestJS 脚手架：** `nest-cli`
- **Java SpringBoot 脚手架：** `Spring Initializr`

你敲一行命令：

```bash
vue create my-project
```

它就自动帮你建好：

- 文件夹结构
- 配置文件（babel、eslint、vite/webpack）
- 基础依赖
- 入口文件、示例页面
- 运行脚本（dev /build）

你**直接写业务代码就行**，不用管环境怎么搭。

## 生活中的比喻

- **盖房子**：脚手架 = 搭好的钢架结构
- **写项目**：脚手架 = 帮你搭好项目结构的工具

你只需要往里面 “砌墙、装修”，不用从零打地基。

## 一句话总结

脚手架 = 快速创建项目模板的工具

帮你省去环境配置，直接开始写业务。

你现在看到的 `vite`、`create-vue`、`nest new` 全都是脚手架。



# 二、为什么要定制脚手架

为什么公司 / 开源项目要自己定制脚手架？

而不是直接用官方的 vue-cli /create-vue？

定制脚手架 = 把公司所有标准、规范、模板、权限、布局、组件，全部打包成一个 “一键生成工具”。

## 1. 统一团队规范

公司里每个人写代码风格不一样。

定制脚手架可以强制：

- 统一目录结构
- 统一 ESLint 规范
- 统一 Git 提交规范
- 统一权限、路由、请求封装
- 统一主题、布局、组件库

**所有人新项目长得一模一样，不会乱。**

## 2. 省去重复搭建

每次新项目都要：

- 装 Vue
- 装 Axios
- 装 Pinia
- 配路由
- 配权限
- 配请求拦截
- 配 ESLint
- 配环境变量
- 配打包配置
- 配布局框架
- 配登录页面

**定制脚手架 = 一行命令全部搞定。**

## 3. 内置企业通用功能

像 **@pureadmin/cli** 这种脚手架，

**内置了企业后台 80% 的通用功能：**

- 登录
- RBAC 权限
- 动态路由
- 按钮权限
- 字典管理
- 布局系统
- 请求封装
- 多主题
- 多语言

**新项目不用从零写，直接二次开发。**

## 4. 降低新人上手成本

新人来了，不用教架构，

**直接运行脚手架，生成标准项目，马上能写业务。**



# 三、开发一个最简脚手架

脚手架 = **命令行工具 + 本地项目模板**

- 命令行工具：负责交互、复制模板、生成项目、安装依赖
- 本地模板：存你标准化的项目架子（Vue3 基础模板，内置在脚手架中，不依赖网络）

## 技术选型

- **Node.js + Commander.js**：处理命令行参数、定义脚手架命令
- **Inquirer.js**：做交互式选择（选模板、选择是否安装依赖）
- **chalk + ora**：美化终端输出（彩色日志、加载动画）
- **fs-extra**：本地文件复制、修改 package.json
- **cross-spawn**：跨平台执行 `npm install`

## 完整实现步骤

### 1. 初始化脚手架项目

```bash
# 1. 创建脚手架项目目录
mkdir my-cli && cd my-cli

# 2. 初始化 package.json
npm init -y

# 3. 安装核心依赖
npm install commander inquirer@8.2.6 chalk@4.1.2 ora@5.4.1 fs-extra@11.2.0 cross-spawn@7.0.3 --save
```

> 说明：固定依赖版本，避免因版本兼容导致 `prompt is not a function` 等报错

### 2. 项目结构

```
my-cli/
├── bin/
│   └── index.js      # 脚手架入口
├── templates/        # 本地模板文件夹
│   └── vue-basic/    # 放入你的 Vue3 项目
├── .gitignore
└── package.json
```

### 3. 配置入口文件 `bin/index.js`

```js
#!/usr/bin/env node

const { Command } = require("commander");
const inquirer = require("inquirer");
const chalk = require("chalk");
const ora = require("ora");
const fs = require("fs-extra");
const path = require("path");
const spawn = require("cross-spawn");

const program = new Command();

program
  .name("my-cli")
  .version("1.0.0")
  .description("我的极简前端脚手架");

program
  .command("create <projectName>")
  .description("创建一个新的前端项目")
  .action(async (projectName) => {
    // 1. 选择模板
    const { template } = await inquirer.prompt([
      {
        type: "list",
        name: "template",
        message: "请选择项目模板：",
        choices: [
          { name: "Vue3 基础模板", value: "vue-basic" },
        ],
      },
    ]);

    // 2. 路径定义
    const templatePath = path.resolve(__dirname, "../templates", template);
    const projectPath = path.resolve(process.cwd(), projectName);

    // 3. 判断目录是否已存在
    if (fs.existsSync(projectPath)) {
      console.log(chalk.red("❌ 项目已存在！"));
      return;
    }

    // 4. 复制本地模板
    const spinner = ora("🚀 正在生成项目...").start();
    try {
      await fs.copy(templatePath, projectPath);
      spinner.succeed("✅ 项目创建成功！");

      // 5. 修改 package.json 项目名
      const pkg = await fs.readJson(projectPath + "/package.json");
      pkg.name = projectName;
      await fs.writeJson(projectPath + "/package.json", pkg, { spaces: 2 });

      // 6. 是否安装依赖
      const { install } = await inquirer.prompt([
        {
          type: "confirm",
          name: "install",
          message: "是否自动安装依赖？",
          default: true,
        },
      ]);

      if (install) {
        const installSpinner = ora("📦 正在安装依赖...").start();
        spawn("npm", ["install"], { cwd: projectPath, stdio: "inherit" }).on("close", () => {
          installSpinner.succeed("✅ 依赖安装完成！");
          console.log(chalk.blue(`👉 启动：cd ${projectName} && npm run dev`));
        });
      }
    } catch (err) {
      spinner.fail("❌ 项目生成失败：" + err.message);
    }
  });

program.parse(process.argv);
```

### 4. 配置 `package.json`

```json
{
  "name": "my-cli", // 注意：发布npm时，需修改为npm上未被占用的名字
  "version": "1.0.0", // 版本号，发布时需递增（如 1.0.1）
  "description": "极简前端脚手架",
  "main": "bin/index.js", // 入口文件，指向bin/index.js
  "bin": {
    "my-cli": "./bin/index.js" // 绑定命令，终端可直接输入 my-cli 执行
  },
  "files": ["bin", "templates"], // 指定发布到npm时要上传的文件夹
  "scripts": {},
  "dependencies": {
    "chalk": "^4.1.2",
    "commander": "^11.1.0",
    "cross-spawn": "^7.0.3",
    "fs-extra": "^11.2.0",
    "inquirer": "^8.2.6",
    "ora": "^5.4.1"
  },
  "keywords": ["cli", "vue3", "前端脚手架", "项目模板"], // 方便npm搜索
  "author": "lining-lo", // 可选，填写自己的名字/昵称
  "license": "MIT", // 开源协议，默认MIT即可
  "private": false // 必须为false，否则无法发布到npm
}
```

### 5. 本地测试脚手架

```bash
# 1. 全局挂载脚手架（把 my-cli 命令绑定到系统，本地测试用）
npm link

# 2. 测试创建项目命令（正常弹出模板选择，即为成功）
my-cli create my-test-project
```

✅ 测试成功标志：能弹出模板选择、无报错、能拉取模板、能安装依赖。

### 6. 发布到 npm

```bash
# 1. 登录 npm（需先在npm官网注册账号，国内用户建议切换官方源）
npm config set registry https://registry.npmjs.org/
npm login # 输入用户名、密码、邮箱（邮箱需验证）

# 2. 发布脚手架（确保package.json中 private: false，name字段无重复）
npm publish

# 3. 发布成功后，团队可全局安装使用
npm install -g 你的脚手架name（如 my-cli）
```

> 踩坑提示：若发布失败，大概率是 name 字段重复，修改 package.json 的 name 后重新发布即可。
